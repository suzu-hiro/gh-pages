主な実績

マインドセット

チーム開発
決められたことしかできないと言うことがありますが、やるべきことが明確に決められない決められたことができていないことも多い認識です。
やるべきことをしっかり認識して成果を出していけば自信に繋がり、そこで初めてそれ以上のことができるようになると思います。
私は、基本設計以降の作業を担当する１メンバーとしてチームに参画することになると思いますので、上記のことを意識して取り組みます。
技術習得
Java、Ecmascriptのように自分の強みである技術については常に、公式サイトで情報収集や必要に応じて実際に動かしてキャッチアップしています。
ライブラリ類については、目新しさや面白さだけではなく、現場で使えるか検討して学習しております。
コーディングスタイル
保守性、分かりやすさ、パフォーマンス、拡張性などを考えて最適なコードを書くように心がけております。
重要なのは、手段を目的化しないことです。新しい技術や概念を学習すると使いたくなるもので、ある程度スキルアップのために必要ですが、影響範囲を慎重に見極めるべきです。
最近javaで、Collectionのメソッドで表現できることを、Stream API を使用するケースが散見されます。計測すると100倍程度差が出る場合もあります。体感できるレベルで問題になることは少ないと思いますが、基本的な考え方として、パフォーマンスを意識してよりシンプルに書くスタイルがよいと思います。
勤怠
体調管理も仕事と考えておりますので、病欠や遅刻・早退はかなり少ない方です。
急病でなければ土日の通院ができる場合も多く、私用も含めて、業務中心にスケジュール管理をします

このサイトについて

使用した技術
基本技術：HTML、CSS、Javascript
font: Font Awesome
ライブラリ： RxJS v6
デザイン
レスポンシブデザインで設計しております。
技術的なポイント
デザインやコーディングは特に他のサイトは参考にしておりません。 極力ライブラリは使用せず、基本的な技術のみを使用しました。
RxJSを使用しているのは、実務では使用したことはありませんが、イベント周りの実装が強力で最近気に入っています。
異なるイベントを一つのストリームとして処理し、豊富なoperatorsで、できたらいいなと思う制御を手軽に利用できるところは、いつか実務で生かしたいです
サイトの狙い
サイトを通じて得意な技術や希望を知っていただき、スキルミスマッチなどにより力が発揮できないことを避けることが目的です
